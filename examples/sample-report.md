## 关键要点（8 条）

1. Python 的 asyncio 基于事件循环模型，适合 I/O 密集型任务
2. 协程（coroutine）是异步编程的基本单位，用 async/await 语法定义
3. 事件循环负责调度协程的执行，一个线程同时只有一个活跃事件循环
4. aiohttp 是最常用的异步 HTTP 库，性能远超 requests
5. 信号量（Semaphore）可以限制并发数，避免对目标服务器造成压力
6. 异步代码中的异常处理需要特别注意，未捕获的异常可能导致静默失败
7. 基准测试显示：100 个并发请求，异步比同步快约 15 倍
8. 不是所有场景都适合异步，CPU 密集型任务应使用多进程
